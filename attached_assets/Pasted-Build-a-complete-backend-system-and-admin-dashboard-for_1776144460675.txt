## DATABASE SCHEMA — ADD THESE TABLES

### orders:
- id uuid primary key default gen_random_uuid()
- order_number text unique (format: SE-2025-0001, auto-increment)
- customer_id uuid references customers
- status text default 'pending'
  values: 'pending' | 'confirmed' | 'sampling' | 'in_production' | 'quality_check' | 'shipped' | 'delivered' | 'cancelled'
- items jsonb (array of order items)
- subtotal_pkr numeric
- discount_percent numeric default 0
- discount_pkr numeric default 0
- total_pkr numeric
- total_usd numeric
- advance_paid_pkr numeric default 0
- balance_due_pkr numeric
- payment_status text default 'unpaid'
  values: 'unpaid' | 'partial' | 'paid'
- payment_method text
- design_image_url text
- design_prompt text
- special_instructions text
- estimated_delivery date
- actual_delivery date
- tracking_number text
- shipping_method text
- whatsapp_updates boolean default true
- created_at timestamptz default now()
- updated_at timestamptz default now()

### customers:
- id uuid primary key default gen_random_uuid()
- name text not null
- email text
- whatsapp text
- country text
- city text
- address text
- company_name text
- total_orders integer default 0
- total_spent_pkr numeric default 0
- customer_type text default 'retail'
  values: 'retail' | 'wholesale' | 'international' | 'vip'
- notes text
- created_at timestamptz default now()

### order_items:
- id uuid primary key default gen_random_uuid()
- order_id uuid references orders
- product_id uuid references products
- product_name text
- category text
- garment_color text
- garment_size text
- fabric text
- gsm text
- quantity integer
- unit_price_pkr numeric
- total_price_pkr numeric
- brand_label boolean default false
- design_image_url text
- design_prompt text
- has_sample boolean default false
- sample_price_pkr numeric default 0

### production_stages:
- id uuid primary key default gen_random_uuid()
- order_id uuid references orders
- stage text
  values: 'order_received' | 'design_approved' | 'sampling' | 'sample_approved' | 'cutting' | 'stitching' | 'printing' | 'embroidery' | 'quality_check' | 'packing' | 'dispatched'
- status text default 'pending'
  values: 'pending' | 'in_progress' | 'completed'
- notes text
- completed_at timestamptz
- created_at timestamptz default now()

### payments:
- id uuid primary key default gen_random_uuid()
- order_id uuid references orders
- amount_pkr numeric
- amount_usd numeric
- payment_method text
  values: 'bank_transfer' | 'paypal' | 'wise' | 'western_union' | 'cash' | 'stripe'
- reference_number text
- status text default 'pending'
  values: 'pending' | 'confirmed' | 'failed'
- notes text
- received_at timestamptz
- created_at timestamptz default now()

### notifications:
- id uuid primary key default gen_random_uuid()
- type text (new_order | payment_received | status_update | new_quote)
- title text
- message text
- order_id uuid references orders
- is_read boolean default false
- created_at timestamptz default now()

### admin_users:
- id uuid primary key references auth.users
- name text
- email text
- role text default 'admin'
  values: 'super_admin' | 'admin' | 'production_manager' | 'sales'
- created_at timestamptz default now()

orders:

  - Customers can only read their own orders (by email match)

customers:
  - Admin only read/write

payments:
  - Admin only

production_stages:
  - Admin read/write, customers read-only for their orders

notifications:
  - Admin only

## API ROUTES (all under /app/api/admin/)

### Orders:
GET /api/admin/orders — list with filters
POST /api/admin/orders — create order
GET /api/admin/orders/[id] — single order with all relations
PUT /api/admin/orders/[id] — update order
DELETE /api/admin/orders/[id] — soft delete (set cancelled)
POST /api/admin/orders/[id]/status — update status + trigger notifications
POST /api/admin/orders/[id]/payment — record payment
GET /api/admin/orders/export — export as Excel

### Customers:
GET /api/admin/customers — list
POST /api/admin/customers — create
GET /api/admin/customers/[id] — detail
PUT /api/admin/customers/[id] — update

### Production:
GET /api/admin/production — all active orders with stages
PUT /api/admin/production/[orderId]/stage — update stage status

### Analytics:
GET /api/admin/analytics/revenue?period=30d
GET /api/admin/analytics/orders?period=30d
GET /api/admin/analytics/products

GET /api/admin/notifications — unread notifications

## AUTOMATED NOTIFICATIONS SYSTEM

### When order is created (from quote submission):
1. Create notification in DB: type 'new_order'
2. Send email to info@signitiveenterprises.com via Resend:
   Subject: "New Order SE-XXXX — [Customer Name] — PKR [Total]"
   Body: full order details, customer info, design thumbnail link
3. WhatsApp notification to admin: +923114661392
   "🔔 New Order! SE-XXXX from [Name] ([Country]). [X] units of [Product]. Total: PKR [Amount]. View: [link]"

### When order status changes:
1. Update order in DB
2. Send WhatsApp to customer (if whatsapp_updates = true):
   Use appropriate template based on new status
3. Send email to customer (if email exists)
4. Create internal notification

### When payment recorded:
1. Update order payment_status
2. Send receipt confirmation to customer via WhatsApp:
   "Payment of PKR [amount] received for order SE-XXXX. Balance: PKR [balance]. Thank you!"
3. Create notification for admin

### Daily digest (cron job via Supabase Edge Function):
Every day at 8am PKT:
- Count: new orders, pending orders, payments due
- Send summary email to admin
- Send WhatsApp summary to admin

## ORDER CREATION FROM STUDIO

When customer clicks "ADD TO QUOTE" in the AI Studio, create a proper order flow:

Step 1 — Review Cart:
- Show all selected items (product, color, size, fabric, GSM, qty, design)
- MOQ progress bar
- Pricing breakdown with discount

Step 2 — Customer Details:
- Name*, WhatsApp*, Email, Country, City, Company (optional)
- Special instructions textarea
- Brand label preference

Step 3 — Confirm & Submit:
- Full order summary
- Two options:
  A) "SEND TO WHATSAPP" — opens WhatsApp with full order details
  B) "SUBMIT ORDER" — creates order in DB, sends email, shows order number
- On submit: 
  - Create customer record (or match existing by WhatsApp)
  - Create order record with status 'pending'
  - Create order_items records
  - Save design image to Supabase Storage
  - Create notification
  - Send emails
  - Redirect to /track with order number pre-filled
  - Show success: "Order SE-XXXX submitted! We'll contact you on WhatsApp within 1 hour."
